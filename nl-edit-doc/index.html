视频非编开发文档

发表于 2017-10-31 | 分类于 前端 | | 文章阅读数：

1、定义或约定

  • 非编面板：非线性编辑面板
       视频条块，或者面板条块，两层含义，既指非编面板上一段一段的意指视频片段的面板条块（表面含义），也指即某播放源的某段播放区间（指代含义）。
  • canvas播放器：canvas播放器播放面板与div控制条的合称
       合成视频：指剪辑完成后，各个视频条块组合而成的新视频。虚概念，页面上没有对应的组件，播放载体是canvas播放器。
       canvas播放器播放面板：简称canvas播放面板。
  • div控制条：canvas播放器控制条，因为是用div做的，就称div控制条
    包括的功能：（1）播放/暂停切换，（2）拖动到某一位置播放，（3）全屏播放/退出全屏播放。
  • 视频源:待做非线性编辑的视频源

2、使用到的两个JS库

  • Seriously.js
    用于实现canvas播放面板，canvas播放面板用于同步视频源的画面。
  • easelJS
    用于实现非编面板，创建视频条块、切割视频条块，移动视频条块，删除视频条块等。

3、初始化工作

  • 视频源部分
    A 获取每个视频源的清晰度、时长信息

  • 非编面板部分
    A 创建和视频源同样数量的视频条块，显示在非编面板中；
    B 将非编信息变量或对象 置0、[]或null等；
    C 非编面板事件监听器的设置；
    D 非编面板中的进度条位置置为0；

  • canvas播放器部分
    A 获取到播放器控制条各个HTML元素的引用
    B 初始化seriously主类

5、交互逻辑

大体上的流程见下图

下面是一些数据逻辑的说明

5.1 非编面板交互interaction

1、初始化工作：开发非编面板的底层工具函数，包括创建面板、创建矩形条块（视频条块）、修改条块、删除条块、创建直线、修改直线位置等。
2、初始化工作：提供各种事件监听，有事件发生就刷新场景。
3、设置三种事件的监听器：
stage.on(“mousedown”, mousedownHandle);
stage.on(“pressmove”, pressmoveHandle);
stage.on(“pressup”, pressupHandle);
4、因为三种事件最后的落脚点都是pressup事件，所以在此区分出三种事件，并在此做非编信息处理逻辑。
5、处理坐标系统的逻辑。包括条块重排，条块越界纠正，拖动条块随鼠标移动等。
6、在视频条块对象上挂载一些可以指代视频源含义的属性，比如videoID，pos，offset等。
7、禁止鼠标右键默认菜单弹出。
8、设置视频条块在拖动时的显示样式（修改颜色和透明度）和修改拖动后的显示层级。

5.2 非编信息处理

将非编信息处理整理成函数nlEditInfoCollect();

1、每次非编交互（删除、拖动、分割条块）后都要调用该函数。
2、函数内部逻辑：
A、计算一共有几个视频条块；B、每个条块是对应哪个视频源的；C、每个条块都多少秒；D、搞一个timeRange数组，存放累加的时间点。E、视频条块总宽度（为了算非编面板上的step）。

5.3 定时器

充当合成视频的时间轴。
主要的逻辑都是一秒（995ms）执行一次。
包括：
1、每秒都判断一次是否达到非编播放的终止条件，达到就执行内部逻辑：停掉所有视频源。
2、每秒都使非编面板上的进度条前进1个step。
3、每秒都判断现在的“秒”是在合成视频的哪个区间段，在哪个区间段播放对应的那个视频源，并在此时将播放画面同步到canvas播放器上。
4、每秒都要判断现在的“秒”是否是关键时间分割点，如果是，需要找到下一个待播剪辑视频段的videoID，并将其的currentTime修改到正确的位置。
5、每秒都要判断是否用户拖动了div控制条上的进度条，如果是：如4，也要修改某个视频源的currentTime。修改非编面板中进度条的位置。并重新将是否拖动进度条的变量设为false。

5.4 非编面板中的进度条

要注意区分该进度条与非编面板中的视频条块；它们是两类东西。
主要是算面板中有多少剪辑的视频条块的时候，可能会误把该进度条也算进去，导致出错。

1、开发两个函数，一个是创建一个直线充当该进度条（createLine），一个是修改一条线的位置保证进度条实例不断前进（modifyLinePos）。
2、初始化时，将进度条位置置0；
3、每播放一秒，进度条前进1个step，这个step = 所有视频条块的宽度和/合成视频的时长；
4、div控制条中的进度条被拖动时，该进度条也要跳转到正确的位置。
5、让非编面板上的进度线始终显示在面板最顶层，不能被视频条块压盖。

5.5 div控制条上的播放/暂停切换

1、根据用户点击图标的名称判断该执行“播放”还是“暂停”的逻辑；
2、如果是播放，A、图标切换为“暂停”；B、div控制条上的总时长赋值；C、每秒刷新已播时间；D、每条刷新进度条上的值。

如果是暂停，A、图标切换为“播放”；B、循环“视频源数组”找出那个在播视频源，停掉它；C、停掉定时器，tick不再增长；D、因为只是暂停，需要记住进度条上的值，赋给定时器的tick变量，待再次点击“播放”时能够续播。

5.6 div控制条上的进度条

使用input range实现

1、点击div控制条上播放按钮时，进度条value要+1；
2、要能判断进度条上的用户拖动事件，拖动后影响：A、合成视频的播放点，要找到当前拖动到的位置是哪个视频源的哪一段的第多少秒的位置；B、非编面板上的进度条的位置。

5.7 div控制条上的全屏/退出全屏切换

基本不涉及数据处理。

0、写好canvas播放器的全屏样式。
1、根据用户点击图标的名称判断该执行“全屏”还是“退出全屏”的逻辑；
2、全屏后，修改div控制条上的图标为“退出全屏”；
3、退出全屏后，修改div控制条上的图标为“全屏”；
4、监听webkitfullscreenchange事件，满足“用户是按ESC退出全屏的而非按了‘退出全屏’按钮退出全屏模式的”逻辑。

deleteShape 删除一个视频条块

modifyRectAlpha 修改视频条块透明度
createLine 创建一条线
modifyLinePos 修改一条线的位置
